openweft

Weft is an open, Go-native cloud platform — microVM-first, multi-hypervisor, multi-tenant, multi-AZ. One binary, server or client mode, like the HashiCorp tools. Without the weft, the warp is just parallel threads.

CGO_ENABLED=0 (pure-Go end-to-end) microVM-first 4 hypervisor backends embedded etcd + Caddy WireGuard mesh per tenant multi-AZ placement BSD-3-Clause
Documentation GitHub

Weft weaves heterogeneous hosts, networks and storage backends into a single cloud platform. weft agent runs the control plane — single-host all-in-one by default, --server/--client split for multi-host — while weft <noun> <verb> is the gRPC client (weft microvm run, weft project create, weft share attach). microVMs are the default execution path : an OCI image becomes a virtio-fs / virtio-9p rootfs on a shared kernel, and weft-microvm-init execs the entrypoint as PID 1.

Pure Go, no cgo. Control plane, agents, in-guest init, FUSE clients, the WireGuard mesh and the BGP router are all CGO_ENABLED=0 — cgo lives only in the isolated Apple-VZ driver plugin. Four hypervisor backends (Apple Virtualization, QEMU/KVM, OpenBSD vmd, Huawei FusionCompute) load as external go-plugin binaries pulled by digest. Cluster state lives in embedded or 3-DC etcd; the reverse proxy is an embedded Caddy (L7 auto-HTTPS + L4); multi-tenant isolation is cryptographic — one WireGuard mesh per Network. Greenfield code is BSD-3-Clause; fork-and-adapt (e.g. weft-block from longhorn-engine) keep their upstream license.

Greenfield code is BSD-3-Clause; fork-and-adapt components keep their upstream license.
